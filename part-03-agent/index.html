把技能与知识库
编排成「项目助理」

前两课分别解决了「怎么做」（技能）和「有什么」（知识库），但它们各自独立。 这节课教你把多个技能和知识库编排成一个统一的项目助理——用户只需要开口说一句话， 项目助理自动判断意图、选对技能、查对知识库、输出结构化回答。

项目助理架构

项目助理 = 统一入口 + 多技能路由 + 知识库增强

用户不需要知道背后有几个技能、几个知识库——只需要一个入口，说清楚自己要什么，项目助理替你分拣、检索、组装。

用户入口

扫码 / 链接 / 嵌入网页——一个统一的对话窗口

意图识别层

用户说了什么 → 应该分到哪个技能？

文书类意图 案例类意图 传播类意图 综合问答

技能路由层

根据意图加载对应技能的 SOP + 输出契约（结项报告 / 项目书 / 案例检索 / 推文...）

知识库 RAG 增强

从云端知识库检索相关文档切片，补充事实、案例、数据，减少幻觉

格式化输出

按技能的输出契约组织回答，标注来源 / 缺失项 / 置信度

升级工作流

按文书链、案例库、传播运营三个模块，各搭建一个多步骤工作流。

互测反馈

扫码互测，每人问 3 个问题，记录「准 / 不准 / 乱答」。

核心思路：先让项目助理「能聊」（步骤 1-2），再让它「聊得好」（步骤 3-4）。不必一次做到完美——第 4 课会按你的主场景继续深化。

三模块工作流

三模块工作流

当简单的「意图 → 技能 → 回答」不够用时，用工作流把多步骤串起来——每个模块都是一条完整的处理链。

模块 M1

文书链工作流

把流水账变成结构化报告的完整处理链。

1
材料预处理：提取关键信息
2
知识库检索：同类报告 + 模板
3
报告生成：六章节输出契约
4
自检 + 缺失项清单
模块 M2

案例库工作流

从需求到案例卡片的精准检索链。

1
意图解析：领域 + 类型提取
2
多维检索：领域 + 类型 + 年份
3
案例卡片：背景→干预→结果→反思
模块 M3

传播运营工作流

从素材到合规发布的内容生产链。

1
素材收集：项目亮点 + 故事
2
内容生成：按平台调整篇幅语气
3
合规检查：脱敏 + 来源校验
互测方法

互测：准 / 不准 / 乱答

和邻座交换项目助理二维码，各问 3 个问题，把回答分成三类——这是发现问题最快的方法。

✅

准

回答正确、格式对、引用了知识库中的真实信息，技能加载也正确。

⚠️

不准

回答了但信息有误，或编造了不存在的数据——说明知识库缺料或检索不到位。

❌

乱答

答非所问、加载了错误的技能、格式混乱——说明意图识别或触发条件有问题。

互测操作流程

  1. 扫码交换：和邻座互扫项目助理的二维码
  2. 各问 3 题：覆盖文书、案例、综合问答等不同场景
  3. 记录分类：每个回答标注「准 / 不准 / 乱答」
  4. 反馈交换：把记录还给对方，课后按反馈修一轮
