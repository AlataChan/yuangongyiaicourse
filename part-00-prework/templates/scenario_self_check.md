# 主场景模块自评清单（M1 / M2 / M3）

> 文件路径：`course/part-00-prework/templates/scenario_self_check.md`
>
> 用法：逐条勾选符合项，哪个模块勾选最多即为你的主场景模块，填入立项书。
> **说不清或并列时，默认 M1 项目文书链**（覆盖面最广，所有机构都要写文书）；第 4 课还可深化第二个模块。

## M1 项目文书链

- [ ] 立项书 / 结项报告 / 资助申请占用团队时间最多
- [ ] 每年要向 ≥2 家资助方提交格式不同的文书
- [ ] 文书常由 1–2 人从零写起，没有可复用模板
- [ ] 对三个模块暂无明确偏好

**对应首选技能**：结项报告生成 或 项目书撰写

## M2 案例与知识沉淀

- [ ] 「经验沉淀 / 案例管理」是机构首要痛点
- [ ] 项目资料散落在云盘、微信群、个人电脑，找不到、难复用
- [ ] 一线服务记录（走访、访谈、活动）很多，但没人整理成案例
- [ ] 同事离职后经验就丢了

**对应首选技能**：案例萃取与检索 或 服务案例脱敏

## M3 传播与运营

- [ ] 活动频次高，每次都要写复盘 / 公众号 / 朋友圈多版本文案
- [ ] 公众号 / 社群运营占用大量人力
- [ ] 社群里重复回答同样的问题（FAQ 场景明显）
- [ ] 传播文风需要保持一致但常走样

**对应首选技能**：活动复盘文案 或 社群 FAQ
