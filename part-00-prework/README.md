## 一、这份作业要解决什么

几乎每家机构都会说「我们想做 AI」——但「做 AI」不是一个可验证的目标。

这份课前作业先用一份短阅读把 **LLM、上下文、知识库、embedding、rerank、Agent、MCP** 等基础概念对齐，再请你把**「我想做 AI」**落到**一个可验证的项目助理场景**，自行填写并提交一份《项目助理立项书》。

它是整个课程的「锚」：第 1 课建技能，第 2 课建知识库，第 3 课组装 AI智能体，全都围绕你在立项书里确认的场景展开。

> 不是考试，没有淘汰。如果某个字段一时填不好，先填能填的，剩下的可在课程社群或可选集中答疑里一起捋清。

## 三、立项书怎么做（4 步）

1. **先完成概念速读**：重点理解上下文窗口、幻觉风险、embedding / rerank、Skill / 知识库 / Agent 的分工。
2. **填主场景模块自评**：用下方第四节判断你的主场景模块（M1 / M2 / M3）。说不清就默认 **M1 项目文书链**。
3. **填立项书模板**：照第五节模板逐字段填写，参考脱敏范例。
4. **第 1 课前提交**：把立项书回填到课程协作空间。未按时提交不影响参加第 1 课，但需课后补交，否则后续环节缺少锚点。

**可选集中答疑（60 min，自愿参加）**：开课前 1 周内安排 1 次线上集中答疑，解答填写疑问、帮拿不准主场景模块的机构当场判断。**这不是 1v1 门诊，缺席不影响开课。**

> 隐私提醒：立项书中**不要填写真实受益人 / 捐赠人 / 志愿者姓名与联系方式**。如需举例，请用 A / B / C 代替。

## 四、主场景模块自评

| 模块 | 推荐条件 | 默认首选技能 |
|------|---------|------------|
| **M1 项目文书链**（默认） | 立项 / 结项 / 资助文书占时最多；或暂无明确偏好 | 结项报告、项目书、资助信 |
| **M2 案例与知识沉淀** | 经验沉淀 / 案例管理是首要痛点；资料散乱难复用 | 案例萃取、服务故事脱敏、案例检索 |
| **M3 传播与运营** | 活动多、公众号 / 社群运营重 | 活动复盘、公众号稿、社群 FAQ |

选模块规则：

1. 痛点明确指向某模块 → 直接选它为主模块。
2. 痛点横跨两个模块 → 选「出错代价更大」或「频率更高」的为主模块，另一个记为扩展模块（第 4 课再做）。
3. 说不清 → **默认 M1 项目文书链**（覆盖面最广，所有机构都要写文书）。
4. 只想做 M3（传播）→ 建议同时挂 M1 或 M2 作为基础，M3 作为进阶扩展。

## 五、《项目助理立项书》模板（A4 单页，复制填写）

```markdown
# 《<机构简称>·项目助理立项书》

## 一句话目标
（30 字内：这个助理帮谁做什么）

## 主场景模块
- [ ] M1 项目文书链   [ ] M2 案例与知识沉淀   [ ] M3 传播与运营

## 输入 / 输出
- 用户：__________  |  输入形式：__________  |  输出形式：__________

## 需要的技能（至少 1 个）
- 技能 1：__________（如：结项报告生成）
- 技能 2（可选）：__________

## 需要的知识库
- 资料类型：__________  |  现有数量估计：__________ 份

## 上线 2 周成功标准
- 量化：__________（如：结项报告从 8h→2h / 案例检索命中率 ≥70%）
- 质化：__________（如：2 位同事愿意日常用）
```

填写要点：

- **一句话目标**要具体到「帮谁、做什么」，避免「让 AI 帮我做事」这类无法验证的说法。
- **成功标准**至少给一个数字（省了多少时间 / 少了几次返工 / 覆盖几个任务）。
- **需要的技能**按「输入 → 输出」拆：一种「输入→输出」的转换 = 一个技能。

## 六、脱敏范例（参考）

```markdown
# 《绿源·项目助理立项书》

## 一句话目标
帮项目负责人把项目执行流水账，变成可直接提交资助方的结项报告。

## 主场景模块
- [x] M1 项目文书链   [ ] M2 案例与知识沉淀   [ ] M3 传播与运营

## 输入 / 输出
- 用户：项目负责人  |  输入形式：项目日志（Word）+ 活动照片清单 + 资助方模板  |  输出形式：六章节结项报告（背景/活动/产出/成效/反思/缺失项）

## 需要的技能（至少 1 个）
- 技能 1：结项报告生成
- 技能 2（可选）：项目书撰写

## 需要的知识库
- 资料类型：历年项目书 + 资助方指南 + 机构模板  |  现有数量估计：约 20 份

## 上线 2 周成功标准
- 量化：结项报告从 3 天缩短到半天，章节无遗漏
- 质化：2 位同事愿意日常使用
```

> 范例中机构名、人名均为占位 / 脱敏，请勿照填真实个人信息。
