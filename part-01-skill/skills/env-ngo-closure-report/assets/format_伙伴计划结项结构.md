# 机构成长支持类结项报告 · 格式参照（脱敏）

> 文件路径：`course/part-01-skill/skills/env-ngo-closure-report/assets/format_伙伴计划结项结构.md`
>
> 本文件说明**正式结项模板**与**本课技能六章节输出**的对应关系。  
> 格式来源于环保公益领域常见的伙伴计划 / 机构成长支持类进展报告，**不含任何真实机构或可识别数据**。

---

## 正式模板常见七大块

| 块序 | 正式模板章节 | 通常写什么 |
|------|-------------|-----------|
| 1 | 项目基本信息 | 区域、周期、预算、执行率、报告人（脱敏后） |
| 2 | 项目目标及完成 | 机构成长目标、个人成长目标、完成情况 |
| 3 | 项目执行进展 | 按产出列活动：时间 / 地点 / 参与人员 / 具体内容 |
| 4 | 后续项目活动安排 | 下阶段计划（可无） |
| 5 | 社群参与情况反馈 | 伙伴网络活动、感受与建议 |
| 6 | 项目风险及其影响 | 执行风险与应对 |
| 7 | 项目传播情况 | 传播篇数、渠道、代表性内容 |

---

## 映射到本技能「六章节输出契约」

| 技能输出章节 | 主要吸收正式模板的哪些块 |
|-------------|------------------------|
| 一、项目背景 | 块 1（基本信息）+ 块 2 摘要 |
| 二、主要活动 | 块 3（活动明细） |
| 三、产出与数据 | 块 3 衡量标准 + 块 2 完成情况中的量化表述 |
| 四、成效与反馈 | 块 2 成效 + 块 5 社群反馈 |
| 五、反思与后续建议 | 块 6 风险反思 + 块 4 后续安排 |
| 六、缺失项清单 | 块 5 / 7 等未量化、未成文的部分 |

> **为什么要压缩？** 正式模板面向资助方归档，字段多；技能输出面向「从流水账快速出初稿」，六章足够覆盖审查要点且便于 QoderWork 角色封装。

## 课堂示范案例（虚构）

- 机构：**青澜环境政策研究中心**（虚构）  
- 项目：**新锐机构成长支持项目（第 1 年）**（虚构）  
- 示范输入：`example_input_流水账.md`  
- 示范痛点：流水账散乱、社群反馈与传播数据未量化——与正式模板要求之间的差距，正是技能要补的。
