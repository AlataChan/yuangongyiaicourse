技能迭代记录卡（每轮 1 条）

文件路径：course/part-01-skill/skills/env-ngo-skill-optimize/assets/iteration_log.md

用法：每轮优化登记 1 条。原则：新失败 → 先加 Gotcha；高频失败 → 同步加评测用例；最后才考虑改 Workflow。

记录模板

### 第 N 轮 · 2026-MM-DD

- 触发原因：评测用例 <ID> 失败 / 真实使用翻车（用户原话：______）
- 病因定位：☐ 触发说明太宽/太窄 ☐ 输出契约不固定 ☐ Gotchas 缺位 ☐ 模板缺失 ☐ 合规
- 修改内容（≤3 处）：
  1. <文件 + 改动，如：SKILL.md description 加「不要用于公众号推文」>
- 优化前 → 后：
  - 前：<旧表现，如：朋友圈文案也触发结项模板>
  - 后：<新表现，如：B1 用例通过，不再误触发>
- 新增评测用例：<ID 或「无」>
- 健康分变化：x/10 → y/10

示例（脱敏）

第 1 轮 · 2026-06-20

  • 触发原因：评测用例 B1 失败（用户原话：「把这段活动记录改成朋友圈文案」）
  • 病因定位：☑ 触发说明太宽
  • 修改内容：
  1. SKILL.md description 末尾加「不要用于公众号 / 朋友圈等传播文案」
  • 优化前 → 后：
  • 前：传播类请求也套用结项报告六章节
  • 后：B1 通过，传播请求不再触发
  • 新增评测用例：B3（「写一条活动预告」）
  • 健康分变化：7/10 → 8.5/10
