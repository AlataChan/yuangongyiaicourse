现场数据台账 · QoderWork 文件夹结构

路径示例：~/项目台账/<项目名称>/

原则：先归集到一个文件夹，再交给 Skill 处理——不在 Skill 里做「跨电脑找文件」。

目录树

<项目名称>_台账/
├── README.md                 # 项目名、周期、站点列表、报告模板说明（1 页）
├── 00_模板/
│   ├── 日报字段说明.md       # 各列含义、单位、谁填
│   └── 周报输出结构.md       # 复制自 weekly_report_template.md
├── 01_原始台账/              # 按站点 + 日期存放，只追加不改动
│   ├── 站点A/
│   │   ├── 2026-06-01_日报.csv
│   │   └── 2026-06-01_照片说明.txt
│   ├── 站点B/
│   └── …
├── 02_周期汇总/              # Skill 输出放这里（可人工改后再定稿）
│   ├── 2026-W23_周报草稿.md
│   └── 2026-06_月报草稿.md
└── 03_已定稿/                # 人工确认后的版本，供知识库入库

文件命名约定

类型格式示例
日报 CSVYYYY-MM-DD_日报.csv2026-06-01_日报.csv
照片说明YYYY-MM-DD_照片说明.txt每行：`时间 \地点 \谁在做什么 \拍摄者`
周报草稿YYYY-Www_周报草稿.md2026-W23_周报草稿.md

日报 CSV 最小字段（可增列，勿删必填列）

日期,站点,活动类型,指标名,指标值,单位,备注,填写人代号
2026-06-01,站点A,垃圾捡拾,捡拾袋数,12,袋,,填写人A
2026-06-01,站点A,宣传接待,接待人数,45,人,,填写人A

QoderWork 归集步骤（课前 / 第 2 课前完成）

  1. 建文件夹：按上树在本机或同步盘创建 <项目名称>_台账/。
  2. 收拢 1 周样本：从各同事电脑复制最近 7 天 CSV + 照片说明到 01_原始台账/<站点>/；文件名统一重命名。
  3. 写 README：列出站点清单、本周起止日期、谁负责哪站。
  4. 在 QoderWork 打开该文件夹：后续 Skill 默认读取此路径；第 2 课可将整个文件夹作为知识库挂载。

与 Skill 的分工

步骤谁做做什么
归集人 + QoderWork文件进统一文件夹、命名规范
汇总env-ngo-field-ledger Skill读周期内 CSV + 照片说明 → 周报/月报草稿
定稿人核对数字、补缺失、移入 03_已定稿/
