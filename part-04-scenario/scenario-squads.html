场景型任务小队样板库

文件路径：course/part-04-scenario/scenario-squads.md

第 4 课配套｜多技能 × 多智能体协作｜公开脱敏版

用法：先看自己的学员指引（MVP），对照下方速查表选一条小队，再填角色卡与交接包。

一、这页解决什么

第 3 课把多个技能挂进一个项目助理；第 4 课要解决的是：多技能同时在线时，如何用多个专责智能体分工、交接、互审，而不是一个助理一口气做到底。

本页提供：

1. 速查表：你的 MVP 类型 → 该用哪条任务小队

2. 三条深写小队：M1 文书 / M2 案例沉淀 / M3 传播（含变体说明）

3. 课堂填空：角色卡、交接包、人审点、5 条测试题干

公开材料只使用脱敏场景类型，不含真实机构名称。个人进度请对照自己的学员指引填写。

讲师展开讲授（翻车对照、话术、巡场）见工作区：teaching/lesson-04-multi-agent/instructor-outline.md。学员跟读：handbook.html 第四节 / 五节。

三、小队 A · M1 文书小队

适用：立项书、结项报告、资方模板填空、PPT 内容大纲。

主模式：流水线协作 + 人机共审。

用户任务（生成结项初稿 / 项目书）
  ↓
主控：判断文体与模板归属
  ↓
资料员：收齐项目书、执行记录、模板、指标表
  ↓
文书员：按资方模板生成初稿（不补无来源事实）
  ↓
指标核对员：对照原始数据核数字与目标达成
  ↓
审核员：查承诺、财务敏感、口径 → 人工确认后才可提交

3.1 角色卡

角色只负责什么必读资料 / 技能输出格式不能做什么
主控判断立项 / 结项 / PPT / 简报，分派下一步项目助理边界卡任务分派 + 交接包不直接写终稿
资料员收齐并索引可用材料，列出缺失项env-ngo-archive-classify、知识库材料清单 + 缺失项不编造缺失数据
文书员按模板出结构完整的初稿机构文书样例、资方模板初稿 Markdown / 大纲不填无来源指标
指标核对员核数字、目标达成、单位一致性原始台账 / 结项数据表核对表 + 差异清单不改叙事口径掩饰误差
审核员查对外承诺、财务敏感、口径风险审校清单问题清单 + 风险等级不替机构提交资方

3.2 交接包示例（文书员 → 指标核对员）

【交接包】
任务目标：核对结项初稿中的成效数字与目标达成表述
输入材料：结项初稿 v1；原始指标表（路径已附）
已确认事实：活动场次、参与人次（来自指标表 sheet「汇总」）
不确定项：第 3 章「社区反馈率」原文未给出分母
输出格式：核对表（字段 / 初稿值 / 原始值 / 是否一致 / 建议）
风险标签：含资助方成效承诺表述
停止条件：原始表缺失关键指标时，标 [待核实] 并停止改写叙事
下一步交给：审核员智能体 → 人工项目负责人

3.3 必停人审点

  • 对外承诺表述（「保证达成」「一定完成」）
  • 财务数字、未公开预算
  • 政策建议中的确定性法规判断

3.4 预制 5 条协作测试（题干）

#类型测试任务
1正常任务「按 XX 资方模板生成结项初稿」→ 主控分到文书链，资料齐全时走完全程
2缺资料缺执行记录 → 资料员列缺失项，文书员不得编造过程描述
3易混任务「写一篇公众号」→ 主控应追问或拒接文书链，导向传播小队
4合规风险初稿出现未公开财务总额 → 审核员拦截，进入人工确认
5返工闭环指标核对失败 → 只回文书员改数字段落，不重做资料搜集

四、小队 B · M2 案例沉淀小队

适用：语音/笔记 → 结构化案例 → 脱敏 → 入库检索。

主模式：流水线协作 + 同伴互审。

用户任务（记案例 / 整理案例 / 入库）
  ↓
主控：判断「整理 / 脱敏 / 萃取 / 入库 / 汇总」
  ↓
记录整理员：忠实转写与字段提取
  ↓
脱敏员：去姓名、地址、可识别信息
  ↓
案例萃取员：四段式案例卡 + 手法标签
  ↓
入库审核员：字段齐全、可检索、无 PII → 人工确认后入库

4.1 角色卡

角色只负责什么必读资料 / 技能输出格式不能做什么
主控判断任务类型与是否需脱敏边界卡分派 + 交接包不跳过脱敏直接入库
记录整理员语音/笔记 → 结构化字段env-ngo-case-extract结构化片段不补原文没有的细节
脱敏员去除可识别信息env-ngo-story-desensitize脱敏稿 + 风险标记不删关键事实来「洗白」
案例萃取员出可复用案例卡与手法标签案例模板四段式案例卡不对外发布
入库审核员检查可入库条件env-ngo-archive-classify通过/驳回 + 原因不放行含 PII 的稿

4.2 交接包示例（脱敏员 → 案例萃取员）

【交接包】
任务目标：将脱敏后的服务片段整理为四段式案例卡
输入材料：脱敏稿 v1（已去除姓名与精确地址）
已确认事实：场馆代称、干预方式、观察到的变化（均来自原文）
不确定项：服务对象年龄原文未提及
输出格式：标题 / 初始状态 / 干预过程 / 关键变化 / 感动瞬间 / 手法标签
风险标签：涉及未成年人服务故事；已脱敏但仍需人审后再对外
停止条件：发现残留真名或精确住址时退回脱敏员
下一步交给：入库审核员 → 人工案例库负责人

4.3 必停人审点

  • 受益人故事是否可对外引用
  • 肖像 / 语音授权是否齐全
  • 未脱敏原文是否误入知识库

4.4 预制 5 条协作测试（题干）

#类型测试任务
1正常任务「把这段现场记录整理成案例」→ 走整理→脱敏→萃取→审核
2缺资料原文只有一句话「孩子变好了」→ 记录整理员标缺失字段，不编造过程
3易混任务「帮我写招募推文」→ 主控导向传播小队，不走案例链
4合规风险输入含真实姓名与手机号 → 脱敏员处理；未脱敏不得入库
5返工闭环入库审核发现残留地名 → 只回脱敏员，不重做萃取全文

五、小队 C · M3 传播小队

适用：公众号、节点宣发、招募包、社群 FAQ、周报转推文。

主模式：主控分派 + 流水线 + 人机共审。

主路径与课堂「青澜公众号任务小队」一致。

用户任务（写推文 / 节点包 / 答疑 / 周报转推文）
  ↓
主控：判断归属（宣发 / FAQ / 复盘 / 周报衍生）
  ↓
资料员 → 结构员 → 写作员 → 审核员 → 人工发布确认

5.1 角色卡（主路径）

角色只负责什么必读资料 / 技能输出格式不能做什么
主控判断任务类型与路由边界卡、触发词表分派 + 交接包不强行猜测意图
资料员搜集事实与来源env-ngo-wechat-research / 定稿周报 / 素材夹素材卡 + 缺失项不编造科普数据
结构员提纲与语气基准env-ngo-wechat-template、历史好稿提纲 + 语气要点不写终稿
写作员生成初稿env-ngo-wechat-humanize / env-ngo-activity-recap初稿 + 配图建议不补无来源事实
审核员隐私、承诺、来源、发布风险env-ngo-wechat-publish、env-ngo-story-desensitize问题清单 + 是否建议人审发布不替机构最终发布

5.2 三条变体（不另开全文，课堂按需选用）

变体何时用链路差异技能重点
主路径活动素材 → 公众号五角色完整流水线微信四件套 + 脱敏
周报→推文已有定稿周报资料员直接读 03_已定稿/ 周报；跳过重复搜集env-ngo-field-ledger → 写作员 → 审核员
节点↔FAQ同时挂宣发与答疑技能主控分派为主：触发词严格区分env-ngo-calendar-campaign ↔ env-ngo-community-faq

节点↔FAQ 路由自检（必做）

用户说法应触发不应触发
「节点是世界河流日，日期…，素材在…」节点宣发包FAQ
「XX 海洋生物是什么」FAQ宣发包
「帮我写活动复盘」活动复盘技能节点宣发

5.3 交接包示例（写作员 → 审核员）

【交接包】
任务目标：发布前检查公众号初稿是否可进入人工确认
输入材料：初稿 v1；素材卡；脱敏规则
已确认事实：活动日期、地点、参与人数（来自活动记录）
不确定项：一张现场照片是否已获肖像授权
输出格式：问题清单 / 风险等级 / 是否建议进入人工发布
风险标签：含受益人故事；含号召性结尾
停止条件：缺核心事实或含未脱敏 PII 时不得建议发布
下一步交给：人工宣传负责人

5.4 必停人审点

  • 科普事实与数据年份
  • 肖像 / 现场照片授权
  • 「直接发布」「不用审核」类请求
  • 对外招募承诺（名额、费用、保障）

5.5 预制 5 条协作测试（题干）

#类型测试任务
1正常任务「把社区减塑活动做成公众号」→ 五角色走完并进入人审
2缺资料缺参与人数 → 资料员列缺失项，写作员不得编造
3易混任务「节点是…」与「XX 是什么」→ 主控分派到不同技能
4合规风险初稿含真实姓名 → 审核员拦截并要求脱敏
5返工闭环审核只指出语气问题 → 回写作员局部修改，不重跑资料搜集

六、课堂 25 分钟：对照样板库拆自己的小队

1. 打开自己的学员指引，确认 MVP 一句话目标。

2. 对照第二节速查表，选定文书 / 案例 / 传播之一（只选一条）。

3. 复制该小队角色表，改成自己机构的说法（仍用脱敏示例数据练习）。

4. 写 1 份相邻角色交接包（八字段齐全）。

5. 圈出 1 个必停人审点，写进协作图。

6. （进阶）从该小队预制测试中挑 5 条，课后实跑并记录。

下课带走一句话：

我的主场景走 _____ 小队，主控交给 _____ / _____ / _____，最后由 _____ 审核收口；人审点在 _____。
