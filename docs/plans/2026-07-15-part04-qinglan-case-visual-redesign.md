## 一、改写目标

将第 4 课原本偏抽象的「一句今晚发」案例，重写为可信的环境公益机构传播任务现场。学员不只记住五个角色名称，而是能解释：

1. 为什么一个看似简单的写稿任务实际包含多种不同判断。
2. 为什么单一智能体容易把「找到一个可用说法」误当成「已经核验」。
3. 多智能体如何通过职责隔离、结构化交接、独立复核和停止条件降低风险。
4. 为什么多智能体的终点仍是「人工确认可发」，而不是自动发布。

## 二、案例设计

### 2.1 真实感来自材料冲突

案例仍使用虚构机构「青澜环境政策研究中心」，但不再只给一句任务和几个事后约束。学员将同时看到以下材料：

| 材料 | 看到的信息 | 冲突 / 风险 |
|------|------------|-------------|
| 宣传同事消息 | 17:20 发来「晚上 8 点前给我可发版」 | 时间压力容易压缩核验和人审 |
| 报名表 | 报名 56 人 | 报名数不等于到场人数 |
| 签到表 | 有 41 个签名，其中志愿者与工作人员未分列 | 「参与者」口径不明，且含真名 |
| 现场群消息 | 有人说「差不多 50 人」 | 口头印象不能替代最终统计 |
| 照片文件夹 | 有可识别儿童正脸 | 尚未找到肖像授权记录 |
| 历史推文 | 语气克制，数字会注明来源 | 不宜改写成「近百人热烈响应」类口号 |

这些数字均为课堂虚构数据，只服务于口径冲突与证据链教学。

### 2.2 从「翻车表现」追到「结构原因」

旧版主要列举模型可能写错什么。新版要进一步说明它为什么会错：

- 写作目标鼓励输出完整、流畅的文章。
- 资料核验要求在证据不足时留白、追问、停止。
- 发布审核要求站在稿件之外找问题，而不是替自己的文章辩护。
- 三种目标同时压在一个上下文里，最容易产生「用一个听起来合理的说法继续写」的捷径。

因此，多智能体的价值不是增加模型数量，而是让不同判断分属不同职责，并让每一步都有可见的输入、输出和退回条件。

## 三、教学结构

1. **收到任务**：17:20 收到「20:00 前给可发版」。
2. **打开材料包**：学员自己指认数字、隐私、授权和语气冲突。
3. **看单一助理如何拉直冲突**：为了交付完整文章，选一个顺口数字、弱化不确定项、把初稿写成终稿口吻。
4. **将判断拆开**：主控、资料核验员、编辑策划员、初稿撰写员、发布审校员分别处理。
5. **走读交接包**：重点看「已确认事实」与「待确认项」如何并存。
6. **看退回而不是硬往下跑**：缺最终口径退回资料核验，隐私风险退回撰写，通过后仍停在人审闸门。

## 四、多智能体原理的表达

| 原理 | 在青澜案例中如何可见 |
|------|----------------------------|
| 职责分离 | 核验员只对证据链负责，撰写员无权改写「待确认」为事实 |
| 上下文隔离 | 每个角色只收到完成本步骤必需的材料和规则 |
| 结构化交接 | 用素材卡、提纲卡、初稿包和审校单传递，不让下游猜上游意图 |
| 独立评估 | 发布审校员依检查清单审初稿，不继承写作员的「这样应该没问题」 |
| 最小返工 | 哪一项不过就只退回负责该项的角色，不把全链条重跑一遍 |
| 人工闸门 | AI 最多交付「发布候选稿」，机构负责人才有最终发布权 |

## 五、图解设计

图像统一使用归藏材质 / Swiss editorial 3D 风格，米白背景、黑色线条、灰白材质、IKB Blue `#002FA7` 为主强调色；审核风险图允许少量 Signal Red `#E60012` 标记阻断点。

### 图 A：「一句今晚发」背后的材料冲突

- 结构：俯视工作台 / 数据场景。
- 对象：催稿消息卡、报名表、签到表、现场照片、历史推文。
- 标签：「时间压力」「人数冲突」「真名风险」「授权待查」。

### 图 B：五角色职责流水线

- 结构：左到右流水线。
- 节点：「主控」→「核事实」→「定结构」→「写初稿」→「审风险」→人工发布闸门。

### 图 C：退回与放行闭环

- 结构：目标与评估器循环。
- 分支：「缺事实」退回核验、「有风险」退回修改、「已通过」进入人审。
- 标签：「缺事实」「有风险」「局部返工」「人审放行」。

## 七、验收标准

1. 学员能从案例材料中自主指出至少 3 个冲突，而不是被动读「翻车点」。
2. 每个智能体都有独立的输入、输出、权限边界和停止 / 退回条件。
3. 文字明确说出职责分离、上下文隔离、结构化交接、独立评估、局部返工和人工闸门。
4. 三张图的中文标签准确、可读，不裁切，无水印与意外英文。
5. 所有网页引用均使用项目内相对路径，无断链。
6. 案例全程使用虚构机构、虚构数据和通用角色称呼，不含现实机构信息。
