知识库 + 工作台知识工作流练习

文件路径：/Users/apple/Documents/4.0 Sanyuan/2.4 环境公益"新"力量/course/part-02-knowledge-base/tools/ima_QoderWork_lab.md

用途：第 2 课初阶练习。目标不是精通某个产品，而是理解“知识库负责资料可检索，工作台负责把知识用于任务”。示例工具可以是 ima、QoderWork、元器应用、Dify 应用或机构已有工作台。

适用范围

适合放入个人知识库 / 工作台不适合放入
个人常读 PDF、网页、公开报告、已脱敏案例L4 敏感信息
需要快速摘要、问答、萃取的微信文件未经授权的内部原始资料
L3 内部资料的个人辅助阅读需要组织共享的正式知识库唯一版本
第 1 课技能运行时常用的个人草稿和参考资料未清洗、无来源、无法确认真实性的资料

课堂任务

  1. 选择 5 份资料：至少包含 1 份 PDF、1 份 Word 或 Markdown、1 份网页或图片资料。
  2. 用 LLM 或知识整理工具先做清洗：
  • 这份资料讲了什么？
  • 哪些事实、数据、案例可以复用？
  • 哪些内容需要脱敏或剔除？
  • 这份资料适合回答哪些问题？
  1. 生成 LLM-wiki 条目：
  • Markdown 正文
  • YAML frontmatter
  • 文档类型、年份、项目名、领域标签
  • 合规档位建议
  • 2–5 个 canonical_questions
  1. 把清洗后的条目放入个人知识库，完成一次问答测试。
  2. 在工作台中调用整理后的资料，跑一次第 1 课技能任务。
  3. 记录一次任务结果：哪些事实被正确引用，哪些仍需要进腾讯元器或继续脱敏。

清洗提示词

请把下面资料整理成适合进入知识库的 LLM-wiki 条目：
1. 删除重复、噪音和与正文无关的内容；
2. 识别并标注可能需要脱敏的信息；
3. 生成 Markdown 正文，保留事实、数据、流程、案例和原始表述；
4. 在开头生成 YAML frontmatter，至少包含 doc_title、doc_type、privacy_level、topic_tags、retrieval_hint、canonical_questions、summary；
5. 如果资料不适合进入组织知识库，请说明原因。

课堂输出

我清洗了 ___ 份资料；
其中 ___ 份适合进入组织层知识库；
___ 份只适合留在个人知识库 / 工作台；
我生成了 ___ 份带 frontmatter 的 Markdown 条目；
我用工作台完成了 ___ 任务；
需要补充或脱敏的资料是：___。

常见误区

误区修正
把个人知识库当正式组织知识库个人知识库更适合资料收纳、阅读和初步萃取，组织共享仍以腾讯元器为主线
资料一导入就上云先做 L1-L4 分级，L2 脱敏审核后再上云
只问“总结一下”要追问“哪些事实可引用、哪些内容需脱敏、适合打什么标签、应该回答哪些问题”
只有正文，没有 frontmatter后续很难做筛选、引用、维护和命中率诊断
只看回答是否顺畅还要检查是否能指出来源和边界

元公益工作室

元公益（北京市公鸣社会工作发展中心）成立于2015年，专注于为公益组织提供战略管理与行业发展的咨询服务，助力其战略管理能力提升、组织发展和转型。同时，以咨询机构和技术支持角色，积极推动公益组织的数字化应用。截止目前，已为超过100家公益组织（包括基金会、民非组织、协会）提供了组织发展、战略咨询或数字化赋能等服务。
