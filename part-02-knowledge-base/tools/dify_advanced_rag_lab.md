> 用途：L2+ 学员或助教使用。Dify 不替代本期主线 Agent平台，主要用于看清 RAG 检索机制和 frontmatter / metadata 对检索质量的影响。

## 定位

| 项目 | 课程口径 |
|------|----------|
| Agent平台 | 本期组织层主线，优先用于项目助理上线 |
| Dify | 高阶训练场，用于理解检索模式、索引、rerank、metadata 过滤 |
| 自建知识库 | 仅作为 L2+ 或后续代建路线，不要求全员掌握 |

## 练习目标

完成一次同资料、同问题、不同检索设置的对比：

1. 上传 5-10 份已脱敏、已清洗、带 frontmatter 的资料。
2. 建立同一知识库的三种测试配置：
   - 向量检索
   - 全文检索
   - 混合检索
3. 如果工具支持 metadata 过滤，尝试使用 doc_type、year、privacy_level、retrieval_hint 做筛选。
4. 使用同一组 10 个问题测试。
5. 记录每种配置的命中率、错误类型和可解释原因。

## 复盘问题

1. 哪些问题全文检索更稳？为什么？
2. 哪些问题向量检索更稳？为什么？
3. 混合检索有没有引入更多无关片段？
4. frontmatter / metadata 过滤是否减少了错误引用？
5. 哪些失败其实不是检索问题，而是资料缺失、清洗不足或提示词问题？
