> 用途：第 2 课组织层主线练习。目标是在腾讯元器中建立可被第 3 课项目助理调用的知识库。

## 上云前检查

| 检查项 | 通过标准 |
|--------|----------|
| 合规档位 | 只包含 L1 和审核后的 L2 |
| 脱敏 | 人名、电话、身份证、住址等已删除或替换 |
| 文件质量 | PDF 无明显乱码，扫描件已 OCR |
| 文件主题 | 一份文档只服务一个主题或项目 |
| frontmatter | 至少有 doc_title、doc_type、privacy_level、retrieval_hint、canonical_questions、summary |
| metadata | 平台内至少能记录文档类型、年份、项目名或领域标签 |

## 练习步骤

1. 新建知识库，命名为「机构简称-知识库」。
2. 建立资料分组：
   - 沟通模板
   - 资助指南
   - 项目管理
   - 案例
   - 志愿者管理
3. 上传 10 份已清洗、已脱敏、带 frontmatter 的 L1/L2 资料。
4. 记录每份资料的 metadata，并确认能对应到 frontmatter 字段。
5. 选择 10 个真实问题做命中率测试。
6. 记录未命中原因，并完成一轮修复。

## 推荐起步参数

| 参数 | 起步建议 | 调整依据 |
|------|----------|----------|
| 切片大小 | 800–1500 中文字或自然章节 | 答案被截断时拆细；上下文不足时略放大 |
| TopK | 3-5 | 找不到资料时提高；引用太杂时降低 |
| 相似度阈值 | 从中等偏高开始 | 乱引用时提高；找不到时降低 |
| 标签过滤 | 按年份、项目、资料类型、privacy_level | 知识库规模变大后必须使用 |
