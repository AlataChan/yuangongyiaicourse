# SG-11 · 公益节点宣发型 · Skill 构建指引（Lite）

> 对应需求代号 T11 | M3 · 主题科普 + 节点日历

---

## 助理画像

| 维度 | 内容 |
|------|------|
| 一句话 | 公益节点日历 + 科普素材 → 短图文/文案 + 倒计时提醒清单 |
| 主模块 | M3 |
| 输入 | 节点主题日历、科普 SOP、历史活动资料 |
| 输出 | 节点专属短图文、宣发文案、提醒清单 |
| 成功标准 | 单节点物料 2h → 0.5h；2 位同事续用；科普准确 |

## 最简 MVP（第 2 课结束前）

**1 个 upcoming 节点** → 从素材文件夹生成 1 套文案（标题+正文+3 条朋友圈短句）+ 1 份提醒清单（T-7/T-3/T-1）。

## 推荐 Skill

| 顺序 | Skill / 资源 | 用途 |
|------|-------------|------|
| 1 | 自建「节点宣发」Skill（Fork activity-recap 改输出契约） | 节点文案 + 提醒清单 |
| 2 | [`content-pipeline-skills`](../part-01-skill/content-pipeline-skills.md) · stop-slop | 去 AI 味 |
| 3 | [`content-pipeline-skills`](../part-01-skill/content-pipeline-skills.md) · wan26-text-to-image | 封面图（可选） |

## QoderWork 三步

1. 建 `节点日历/`：`YYYY-MM-DD_节点名.md` 列表 + `科普素材/` 子文件夹。
2. 写节点 Skill 触发说明：「用户提供节点名称+日期时加载，不要用于日常活动复盘」。
3. 用最近 1 个节点试跑，人工核对科普事实。

## 分课卡点

| 课次 | 做什么 |
|------|--------|
| 第 1 课 | 节点宣发 Skill + 固定输出结构（标题/正文/短句/提醒表） |
| 第 2 课 | 科普素材入库 |
| 第 4 课 | 第二 Skill：community-faq（社群常见科普问） |
| 第 5 课 | 展示下月 2 个节点的预生成包 |

## 常见坑

- 科普事实无出处 → 输出标注「请核对来源」；敏感数据不写死。
- 节点 Skill 与日常 recap 冲突 → 触发写清「仅节点日历触发」。
- 只做文案不做提醒 → 提醒清单是 MVP 一部分。
