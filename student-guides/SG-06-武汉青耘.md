## 一、📋 助理画像

**一句话目标**：核心不是"做很多事"，而是"先把散落资料整理成可用的体系"。

| 维度 | 内容 |
|------|------|
| 主模块 | M2 案例与知识沉淀 + M3 传播与运营 |
| 谁来用 | 项目负责人、项目主管、实习生 |
| 输入什么 | 方案/反馈/影像/总结/传播等散落文件 |
| 输出什么 | 按五分类整理的项目库、年终总结、公众号草稿等 |
| 现有资料 | 约 40 份项目资料，全在个人电脑和移动硬盘 |
| 成功标准 | 1 个项目按五分类整理完毕；真实减少项目主管重复劳动 |

## 三、🎯 最简闭环 MVP

> 第 2 课结束前必须跑通。⚠️ 只做一个任务，其余后置。

**MVP 定义**：把 1 个项目的所有文件放进文件夹 → QoderWork 按五分类自动重命名+归档 → 生成目录索引。

**为什么选这个、不选其他的**：
- 这是你自己写的唯一量化成功标准："1 个项目按方案/反馈/影像/总结/传播分类一遍"
- 资料整理是所有后续功能（年终总结、公众号、志愿者库）的前提
- 这个跑通了，后续所有的"生成"才有地基

**跑通长这样**：
```
输入：文件夹中有 20+ 个乱序文件（.docx, .jpg, .xlsx, .mp4, .pdf...）
QoderWork 执行：
  1. 读取每个文件的内容（非纯靠文件名）
  2. 判断属于哪个分类（方案/反馈/影像/总结/传播）
  3. 重命名为 "[项目名]_[分类]_[日期]_[原标题]"
  4. 按分类移动到对应子文件夹
  5. 生成一个目录索引文档，列出所有文件及其分类
```

## 四、🧩 Skill 模板

### Skill 1：项目库五分类归档（第 1-2 课，必做）

```
【Skill 名称】项目资料五分类归档
【触发方式】输入 "整理项目 [文件夹路径]"
【输入】一个含该项目所有散落文件的文件夹
【处理逻辑】
  1. 逐个读取文件内容和文件名
  2. 按以下分类规则判断归属：
     - 方案类：项目计划书、活动方案、预算表、申报材料
     - 反馈类：参与者问卷、甲方/资方反馈、评估报告
     - 影像类：照片、视频、截图、海报
     - 总结类：活动总结、结项报告、复盘记录
     - 传播类：公众号链接、新闻稿、朋友圈文案
  3. 重命名模板：[项目名]_[分类]_[日期]_[原文件名保留关键信息]
  4. 创建五个子文件夹，移动文件
  5. 生成索引目录
【输出格式】五个子文件夹 + 一份目录索引文档
【质检标准】每个文件分类正确；原文件内容无损；索引可点击/可搜索
```

### Skill 2：年终总结生成（第 3 课，分类完成后自然产生）

```
【Skill 名称】年终项目总结
【触发方式】输入 "生成年度总结 [年份]"
【输入】该年所有已整理项目文件夹
【处理逻辑】读取全年各项目总结→按"核心成就/关键挑战/经验教训/明年计划"汇总
【质检标准】每个项目至少 1 个量化产出；总结不重复、不遗漏
```

### 复制即用

```
你是一个项目资料整理助手。请按以下规则工作：

1. 当我输入"整理项目 [文件夹路径]"，请逐一读取该文件夹中每个文件的内容和文件名。

2. 按以下五个分类判断每个文件的归属：
   - 方案类：项目计划书、活动方案、预算表、申报材料
   - 反馈类：参与者问卷、甲方/资方反馈、评估报告
   - 影像类：照片、视频、截图、海报
   - 总结类：活动总结、结项报告、复盘记录
   - 传播类：公众号链接、新闻稿、朋友圈文案

3. 判断规则：
   - 优先根据文件内容判断（而非仅靠文件名）
   - 如果一个文件同时符合多个分类，选最核心的那个
   - 无法判断的单独放入"待分类"文件夹并标注原因

4. 重命名格式：[项目名]_[分类]_[日期]_[原标题核心信息]

5. 最后生成一个索引文档，列出所有文件的分类、路径和一句话摘要。

现在请开始执行。
```

## 五、🛠️ 分课操作指引

### 第 1 课：验证分类规则 → 跑通一个项目

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：一个测试项目的分类结果 | ✅ 完成标志：所有文件分类正确率 ≥80%

1. 把课前准备的测试项目文件夹路径告诉 QoderWork
2. 粘贴"复制即用"提示词 → 输入：`整理项目 [文件夹路径]`
3. QoderWork 开始读取→分类→重命名→生成索引
4. 人工检查每一个分类是否正确（这是最关键的一步）
5. 如果某个文件分错了，告诉 QoderWork 原因 → 让它调整分类规则

### 第 2 课：封装 Skill + 扩展到全部项目

> ⏱ 课上 15 min + 课后 25 min | 📤 提交物：五分类归档 Skill + ≥10 个项目库 | ✅ 完成标志：输入触发词可整理任意项目文件夹

1. 封装 Skill，命名为"青耘项目库整理"
2. 用第二个项目测试
3. 把其余项目分批整理（每次 2-3 个，不要一次性全扔进去）
4. 准备一份"青耘 AI 助理使用 1 页纸"给同事

### 第 3 课：年终总结 + 公众号草稿

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：一份年终总结初稿 + 一篇公众号草稿 | ✅ 完成标志：总结涵盖了所有已整理项目

1. 项目库整理完成后，用 Skill 2 生成年度总结
2. 选一个有传播价值的项目 → 输入关键信息 → 生成公众号草稿（内容优先，排版后调）
3. 对比：你手工写一份年终总结需要多久？

### 第 4 课：志愿者库 + 甲方产品库（按精力扩展）

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：志愿者库整理方案或产品展示页 | ✅ 完成标志：至少启动一个辅助 Skill

1. **志愿者库**：让 QoderWork 整理群聊中的志愿者信息，提取"姓名/技能/参与项目/活跃度"
2. **甲方产品库**：挑选 3-5 个最有代表性的项目，生成展示页面

### 第 5 课：成果预发布

> ⏱ 课上 15 min | 📤 提交物：面向同事的使用图文指南 | ✅ 完成标志：项目主管独立使用了 1 次

1. 演示"青耘项目库整理"Skill 的完整操作
2. 展示一个项目从"乱序文件"到"五分类归档"的全过程
3. 收集同事反馈：最容易上手的功能是什么？最想加的功能是什么？

## 六、⚠️ 常见坑提醒

| 坑 | 自救方法 |
|----|---------|
| 40 份资料一次喂进去 AI 搞乱 | 分批次处理：每个项目单独整理，一次不超过 2 个项目 |
| 文件分类总出错（影像类被分到总结类） | 检查文件格式：.jpg/.mp4 强制归影像；.docx 才看内容判断 |
| 忍不住想同时做 7 个功能 | **强制收敛**：第 1-3 课只做分类归档，其余功能放在第 4 课再评估；完成一个再开工下一个 |
| 资料在个人硬盘，没有备份 | ⚠️ 第 0 步一定是备份。AI 操作文件前，确保原始文件已有完整副本 |
