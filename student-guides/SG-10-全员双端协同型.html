SG-10 · 全员双端协同型 · Skill 构建指引（Lite）

对应需求代号 T10 | M3 · 全员办公提效 | L1 正课 / L2 集成选修

助理画像

维度内容
一句话全员可用的项目助理：文书 + 宣传 + 答疑 + 案例，先单端跑通再谈双端
主模块M3（兼 M1/M2 能力）
输入工作文本、项目资料、活动素材、协作软件表单导出
输出标准文书、宣传物料、FAQ、案例、复盘
成功标准L1：元器 Bot 稳定可用，文稿时长 -60%；L2：自有平台+协作软件双端（选修）

最简 MVP（第 3 课结束前）

元器 Bot + 1 个知识库（≥30 份核心资料）+ 2 个 Skill 跑通三类任务各 1 次：文书框架、活动文案、资料检索问答。

推荐 Skill

顺序Skill用途
1env-ngo-proposal-draft 或 closure-report文书初稿
2env-ngo-community-faq统一答疑话术
3env-ngo-activity-recap宣传物料
L2 选修平台 API / 协作软件插件文档双端内嵌，工作坊或 office hour

QoderWork 三步

  1. 从硬盘+协作网盘先收 30 份高频资料到统一文件夹（分批，每批 ≤2GB）。
  2. 第 2 课建知识库索引；定义 10 条标准测试问句。
  3. 第 3 课挂载到元器；L1 验收到此为止。

分课卡点

课次做什么
第 2 课知识库分批入库 + 检索测试
第 3 课Agent 编排 + 三类任务冒烟测试
第 4 课第二 Skill + 路由边界（文书 vs FAQ vs 推文）
第 5 课内测：≥3 位同事各完成 1 次真实任务
工作坊（L2）双端集成方案评审，不纳入 5 课必修

常见坑

  • 一上来做 API 集成 → 先 L1 元器跑通再扩展。
  • 1000 条资料一次入库 → 先 30 份核心，验证再扩。
  • Skill 过多路由冲突 → 第 4 课必做路由自检表。

元公益工作室

元公益（北京市公鸣社会工作发展中心）成立于2015年，专注于为公益组织提供战略管理与行业发展的咨询服务，助力其战略管理能力提升、组织发展和转型。同时，以咨询机构和技术支持角色，积极推动公益组织的数字化应用。截止目前，已为超过100家公益组织（包括基金会、民非组织、协会）提供了组织发展、战略咨询或数字化赋能等服务。
