## 一、📋 助理画像

**一句话目标**：把散落在多人电脑、移动云盘、钉钉文档、U 盘里的草原垃圾捡拾/边境巡护/野生动物登记台账，收进统一文件夹，AI 帮汇总成每周项目报告初稿；同一批素材再自动生成适配小红书/抖音/公众号的传播内容。

| 维度 | 内容 |
|------|------|
| 主模块 | M2 案例与知识沉淀（+M3 传播） |
| 谁来用 | 项目主管（长期固定）+ 轮换志愿者（短期） |
| 输入什么 | 4 站点每日台账 Excel（垃圾捡拾及分类/边境巡护/迁徙野生动物登记）+ 工作照说明（谁在做什么+拍摄者）+ 活动执行记录 |
| 输出什么 | 周报/月报/执行报告初稿；小红书/抖音/公众号推文及视频脚本 |
| 现有资料 | 每日 4 份 Excel（近半年）；工作照每天约 80M × 4 站点；钉钉云文档 + U 盘 + 个人电脑分散存储 |
| 成功标准 | 周报撰写 2h→≤0.5h；2 周内产出 1 篇可直接排版推文；1 套可落地视频脚本；操作简单到志愿者可独立上手 |

## 二、📦 课前准备清单

> ⏱ 预计 40 分钟 | 第 1 课前完成

- [ ] **建统一台账文件夹**：在一台「最稳定的设备」（优先办公电脑）新建 `草原台账/01_原始台账/`，建子文件夹：`站点甲/`（边境巡护）、`站点乙/`（垃圾捡拾）、`站点丙/`（野生动物登记）、`站点丁/`（其他）
- [ ] **收拢 1 周样本**：从 U 盘、钉钉、移动云盘各取最近 7 天内最全的那 4 份 Excel，复制进对应站点子文件夹，统一改名为 `YYYY-MM-DD_日报.xlsx`
- [ ] **整理照片说明模板**：制作示例一行：`日期 | 站点 | 志愿者A在边防路X公里处捡拾垃圾+分类 | 志愿者B拍摄`，存为 `照片说明模板.txt` 发进工作群
- [ ] **确认台账列名一致性**：4 个站点的 Excel 是否有相同列名？（如「捡拾袋数」在有些表里写「袋」，有些写「包」，需统一）
- [ ] **工具确认**：QoderWork 已安装；LTE/网络在工作站点是否稳定（高原/边境地区需提前确认）

## 三、🎯 最简闭环 MVP

> 第 2 课结束前必须跑通

**MVP 定义**：用 2 个站点 × 7 天台账，跑通 `env-ngo-field-ledger`，产出 1 份周报草稿（数据可溯源、缺报标出）。

**与 SG-09 的关键区别**：
- 二连浩特台账类型更多（捡拾/巡护/野生动物三类并行），列名需提前统一
- 数据来源更分散（移动云盘 + 钉钉 + U 盘），收拢步骤是最大阻力点，需前置解决
- 传播场景包含小红书「日常记录」风格，语气轻松；抖音还需口播脚本格式

**跑通长这样**：
```
菅瑞民：请用 01_原始台账/站点甲、站点乙/ 的 6.23-6.29 台账和照片说明，
        生成本周周报初稿。

AI 助理输出：
---
草原巡护项目周报（2026-06-23 至 2026-06-29）

一、本周工作概述
  本周在站点甲（边境巡护）和站点乙（垃圾捡拾）共完成……

二、分站点数据汇总
  | 活动类型 | 站点 | 指标 | 本周合计 | 数据来源 |
  | 垃圾捡拾 | 站点乙 | 捡拾袋数 | 47 袋 | 0623-0629日报 |
  | 边境巡护 | 站点甲 | 巡护线路次数 | 5 次 | 0623-0629日报 |
  | 野生动物 | 站点甲 | 迁徙记录条数 | 待核对 | 0625日报缺失 |

三、数据核对说明
  ⚠ 站点甲 0625 日报缺失，野生动物登记条数 [待补录]

四、缺失项清单
  - 站点甲 2026-06-25 日报（请负责人补报）
---
```

## 四、🧩 Skill 模板

### Skill 1：台账→周报（第 1-2 课，主 Skill）

```
【Skill 名称】草原巡护周报生成
【触发方式】输入「生成周报 [起止日期]」
【输入】
  - 台账文件夹路径（01_原始台账/<站点>/）
  - 报告周期（起止日期）
  - 可选：历史周报样本（对齐格式）
【处理逻辑】
  1. 按站点 × 活动类型分类汇总（垃圾捡拾/边境巡护/野生动物/其他各一栏，不混合）
  2. 匹配照片说明到对应条目；说明格式不符的标注「照片说明格式待规范」
  3. 缺报站点/日期写入「缺失项清单」，不估算
  4. 不同单位（袋/公斤/条）分列展示
【输出格式】Markdown，按 weekly_report_template.md 结构
【质检标准】每项数据有 Excel 文件来源；缺报全部标出；不跨文件夹找数据
```

### Skill 2：周报素材→传播内容（第 4 课）

```
【Skill 名称】草原传播生成
【触发方式】输入「写推文 [平台]」（与「生成周报」触发词明显区分）
【输入】
  - 最近 1 份定稿周报（从 03_已定稿/ 读取）
  - 目标平台（公众号/小红书/抖音口播脚本）
  - 可选：本期突出亮点（如「本周发现迁徙XX数量创近年记录」）
【处理逻辑】
  - 公众号：800字，有引人入胜的野外细节；科学数据有来源
  - 小红书：300字+5个标签（草原/内蒙古/公益/野生动物/边境等相关）；口语化，有生活感
  - 抖音脚本：分镜格式（序号/画面描述/口播文字），时长参考 60-90 秒
  - 人名全部脱敏为代号；结尾加 [发布前请确认肖像授权]
【输出格式】各平台分块输出，单文件 Markdown
```

### 复制即用（在 QoderWork 中粘贴）

```
你是草原巡护项目的数据助手，负责整理台账、生成周报和传播内容。请严格按以下规则工作：

1. 当我输入「生成周报 [日期范围]」：
   - 读取指定路径下各站点 Excel，按「垃圾捡拾/边境巡护/野生动物/其他」分类汇总
   - 不同单位（袋/公斤/条/人次）分开展示，严禁混加
   - 缺报站点/日期全部标入「缺失项清单」，不估算、不编造
   - 输出结构：概述 → 分站点汇总表 → 活动记录 → 数据核对说明 → 缺失项清单

2. 当我输入「写推文 [平台]」：
   - 从最近定稿周报中提取画面感最强的野外场景作为核心内容
   - 公众号800字有引子；小红书300字+标签；抖音分镜口播60-90秒
   - 人名改为代号A/B/C；结尾加[发布前请确认肖像授权]
   - 传播内容的数据必须来自定稿周报，不额外编造

3. 任何时候不要猜测我没提供的数据。
```

## 五、🛠️ 分课操作指引

### 第 1 课：台账收拢 + 首次周报

> ⏱ 课上 15 min + 课后 35 min | 📤 提交物：2 站点 1 周的周报草稿 | ✅ 完成标志：各类活动分站点汇总，缺报有标注

1. 把课前准备好的 2 站点 7 天数据放入统一文件夹（重点：4 站点 Excel 列名是否已统一？）
2. 粘贴「复制即用」提示词，触发「生成周报 2026-06-23 至 2026-06-29」
3. 检查：垃圾捡拾和野生动物数据有没有混在一起？缺报的站点/天标出来了吗？
4. **重要课后任务**：把另外 2 个站点的台账也收进来；制作照片说明模板发工作群，统一格式

### 第 2 课：4 站点全量跑通 + 定稿入库

> ⏱ 课上 20 min + 课后 25 min | 📤 提交物：完整 4 站点 1 周周报草稿 | ✅ 完成标志：三类活动数据分清，无单位混淆

1. 补入另外 2 站点台账，跑完整 4 站点周报
2. 重点检查「袋/公斤/条」是否混加了，及时纠正
3. 人工核对后定稿，移入 `03_已定稿/`
4. 知识库试挂：看过去 1 个月定稿周报，能查「5 月份站点乙捡拾总量」吗？

### 第 4 课：传播 Skill + 平台路由

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：1 篇公众号稿 + 1 条小红书稿 | ✅ 完成标志：小红书口语化有生活感，公众号有野外细节

1. 用本周定稿周报触发「写推文 公众号」，再触发「写推文 小红书」
2. 对比两版风格差异：公众号够科学吗？小红书够轻松吗？
3. 试跑抖音脚本：看分镜格式是否清晰，画面能否拍到
4. 路由自检：「生成周报」vs「写推文」触发词不能混用

### 第 5 课：志愿者移交 + 成果演示

> ⏱ 课上 20 min | 📤 提交物：1 页《志愿者使用说明》（1 张 A4，含触发词+文件夹路径） | ✅ 完成标志：1 名新志愿者看说明能独立跑通周报

1. 演示完整流程（录台账 → 生成周报 → 生成推文 → 5 分钟内完成）
2. 着重展示：新志愿者流动时，只要按同一个文件夹命名规范，就能无缝接手
3. 打印 1 页说明书贴在办公室：触发词是什么 + 文件放哪里 + 结果在哪里找
4. 可选：与 SG-09 学员互测，交换各自台账试跑，相互找问题

## 六、⚠️ 常见坑提醒

| 坑 | 自救方法 |
|----|---------|
| U 盘/钉钉/移动云盘多源数据，收拢时命名冲突 | 先按「站点-日期-类型」重命名再合并；同一天同一站点合为一份 Excel |
| 4 站点 Excel 列名不统一（「袋」vs「包」vs「条」） | 课前做一份「列名对照表」贴在文件夹里；后续统一用标准列名 |
| 照片说明格式乱（有人写句子，有人只写地名） | 统一用模板：`日期|站点|谁在做什么|拍摄者代号`，置顶工作群 |
| 周报数字准了但传播稿没亮点 | 在照片说明里找：有没有「发现迁徙数量异常多」「志愿者在极端天气坚持」这类画面感场景？用那个做开头 |
| 网络差导致 QoderWork 响应慢 | 课程期间操作尽量在 WiFi 条件下完成；演示用事先截图/PDF 备用 |
