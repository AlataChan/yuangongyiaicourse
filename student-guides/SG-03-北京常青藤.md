# SG-03 · 北京常青藤 · Skill 构建指引

> 李方方（所长）| M2 · 案例与知识沉淀 | ⭐ 最独特场景：语音→案例

---

## 一、📋 助理画像

**一句话目标**：一线执行人员语音讲述"受益对象变化+做了什么"→ AI 自动整理成文字案例。

| 维度 | 内容 |
|------|------|
| 主模块 | M2 案例与知识沉淀 |
| 谁来用 | 3 个场馆馆长、一线执行人员 |
| 输入什么 | 一线人员语音口述；已有 20+ 案例文本 |
| 输出什么 | 结构化成效故事案例（文字）、服务手法梳理 |
| 现有资料 | 20+ 已有案例（2 成在云盘），100+ 未收集（6 成在一线服务伙伴手里） |
| 成功标准 | 跑通"收集→生成"全流程；3 个场馆馆长试用并愿意继续使用 |

## 二、📦 课前准备清单

> ⏱ 预计 40 分钟 | 第 1 课前完成

- [ ] **精选已有案例**：从云盘和伙伴处收集 5-8 个最完整的服务案例文本，放入文件夹 `常青藤_案例素材/`
- [ ] **拆解你心目中的"好案例"**：选一篇你最满意的案例，标注它的结构是什么（如：孩子初来时的状态 → 我们做了什么 → 一个月后的变化 → 一句话的感动瞬间）
- [ ] **工具确认**：
  - QoderWork 已安装
  - 企业通讯工具 Chatbot 已绑定 QoderWork（用于语音输入）
  - 手机端企业通讯工具测试：发一条语音给 Chatbot，确认能接收
- [ ] **找一位场馆馆长**简单聊一聊：如果每次活动后只需要花 2 分钟说一段语音、不用写，ta 愿意试试吗？

## 三、🎯 最简闭环 MVP

> 第 2 课结束前必须跑通

**MVP 定义**：一线人员用手机发一条语音到 Chatbot → QoderWork 转文字 → 按案例模板结构化输出。

**为什么选这个**：
- 从"语音→文字"开始，不跳步做"文字→案例"
- 先验证一线人员"愿意开口说"这个最大前提
- 模板可以先用已有的 5-8 个案例精炼出来

**跑通长这样**：
```
馆长语音输入（2 分钟）：
"今天下午在小武基场馆，有个叫 A 的男孩，来的时候特别不配合，
一直蹲在角落。我们老师就陪着他画了 40 分钟画，慢慢开始说话了。
走的时候他还主动跟老师说再见。我觉得变化挺明显的。"

QoderWork 输出（<30 秒）：
【服务片段】2026.6.20 · 小武基场馆
- 服务对象：男孩 A
- 初始状态：不配合，蹲角落不沟通
- 干预方式：老师陪伴绘画 40 分钟
- 即时变化：开始说话，主动道别
- 一句话感动：从一开始拒绝到主动道别，只用了一幅画的时间
```

## 四、🧩 Skill 模板

### Skill：语音成效故事收集（第 1-5 课持续迭代）

```
【Skill 名称】常青藤成效故事收集
【触发方式】企业通讯工具 Chatbot 发送语音 + "记一下" 触发词
【输入】一线人员的语音口述（1-3 分钟）
【处理逻辑】
  第 1 步：语音转文字，保留原文
  第 2 步：按模板提取结构化字段
     - 日期/场馆
     - 服务对象（脱敏代称）
     - 来时的状态
     - 老师做了什么
     - 发生的变化
     - 一句话感动
  第 3 步：按场馆分类存入资料库
  第 4 步（第 4 课后）：每周汇总一次，识别共性服务手法
【输出格式】结构化文本片段，存入对应场馆文件夹
【质检标准】转写文本准确；结构化提取不丢失关键描述；不使用真实姓名
```

### 案例模板（基于已有案例精炼）

```
【案例标题】[场馆]-[日期]-[一句话主题]
【服务对象】[代称]，[年龄]，[来馆时长]
【初始状态】3-5 句，描述来馆时的情况
【干预过程】3-5 句，描述老师做了什么
【关键变化】3-5 句，描述观察到的变化（行为/情绪/社交/语言）
【感动瞬间】1 句话，最有触动的那一刻
【服务手法标签】[绘画陪伴] / [情绪疏导] / [同伴互动] 等
```

### 复制即用（在 QoderWork 中粘贴此段来生成 Skill）

```
你是一个帮助公益机构收集一线服务案例的助手。

当收到一段关于儿童服务的语音转录文本后，请按以下结构整理：

1. 基本信息：日期、场馆、服务对象代称
2. 初始状态：孩子来馆时的表现（行为、情绪）
3. 干预过程：老师具体做了什么（陪伴方式、用了什么工具/方法）
4. 关键变化：观察到的变化（至少 1 条具体描述）
5. 一句话感动：最打动你的那个瞬间
6. 服务手法标签：从以下标签中选择 1-2 个：[绘画陪伴] [户外自然] [情绪疏导] [同伴互动] [阅读引导] [其他]

要求：
- 忠实于原文，不添加原文没有的细节
- 服务对象使用代称（A/B/C），不使用真实姓名
- 不确定的信息标注 [待核实]
- 如果原文信息不足以填满模板，只填写有信息的字段

现在请建立此规则，按每条语音转录逐一处理。
```

## 五、🛠️ 分课操作指引

### 第 1 课：跑通"语音→文字"管道

> ⏱ 课上 15 min + 课后 15 min | 📤 提交物：第一条语音→文字的完整记录 | ✅ 完成标志：语音转文字准确率≥90%，基础结构化可识别

1. 在学生指南中找到已精炼的案例模板，复制到 QoderWork
2. 用一条已有案例的文字版测试：粘贴原文 → 让 QoderWork 按模板结构化
3. 调整模板中不适用的字段（如"服务手法标签"是否覆盖你们的实际手法）
4. **课后**：请一位场馆馆长用手机发一条测试语音到 Chatbot → 查看转文字效果 → 用模板结构化

### 第 2 课：封装 Skill + 语音全链路跑通

> ⏱ 课上 20 min + 课后 20 min | 📤 提交物：语音→结构化案例的完整 Skill | ✅ 完成标志：馆长发语音→30 秒内拿到结构化片段

1. 把第 1 课优化后的提示词封装为 Skill
2. 设置触发词（如"记一下"），参考"随手记→晚上整理"的工作流
3. 测试：馆长发一条语音 → QoderWork 自动处理 → 查看输出
4. 收集馆长反馈："这个速度你能接受吗？""缺了什么信息？"

### 第 3 课：案例库搭建

> ⏱ 课上 15 min + 课后 30 min | 📤 提交物：≥10 个案例的结构化资料库 | ✅ 完成标志：可按"场馆/服务手法标签"搜索

1. 把已有 20+ 案例逐一用 Skill 结构化（先用文字版，不用语音新增）
2. 按场馆分子文件夹存放
3. 建立标签索引：服务手法标签覆盖了哪些？
4. 测试搜索："所有用过绘画陪伴的案例有哪些？"

### 第 4 课：批量收集 + 初步汇总

> ⏱ 课上 15 min + 课后 25 min | 📤 提交物：一周语音收集 + 周报汇总 | ✅ 完成标志：识别出至少 2 条共性服务手法

1. 3 个场馆馆长每天各发 1-2 条语音，持续一周
2. 周末让 QoderWork 汇总本周所有案例 → 识别最常出现的服务场景
3. 提取共性手法："哪些方法最常被提到？"→ 这就是你们的知识资产

### 第 5 课：成果预发布

> ⏱ 课上 15 min | 📤 提交物：向资助方展示的一份成效案例集 | ✅ 完成标志：至少 1 个场馆馆长在日常工作中持续使用 ≥1 周

1. 输出一份"本月 10 个最打动人心的故事"案例集
2. 馆长演示：现场说一段语音，展示从语音到案例的完整链路
3. 收集定性反馈："这个案例比你自己写的，少花多少时间？"

## 六、⚠️ 常见坑提醒

| 坑 | 自救方法 |
|----|---------|
| 馆长不愿意开口说（习惯写或者不好意思） | 先让所长带头示范，录第一条；强调"说 2 分钟就行，不用管语法" |
| 语音转文字准确率低（口音/环境噪音） | 建议用手机自带录音转文字 → 复制文本发给 Chatbot 作为过渡方案 |
| 6 成资料在一线伙伴手里收不上来 | 这不是技术问题。先跑通"新增收集"管道（日常语音），再慢慢消化存量 |
