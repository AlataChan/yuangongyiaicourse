# SG-08 · 天津绿邻居 · Skill 构建指引

> 安旭（主任）| M2+M3 · 知识库问答 | ⭐ 唯一提出智能问答需求

---

## 一、📋 助理画像

**一句话目标**：把 50 个项目资料变成可搜索、可问答的内部知识库，并支持结项报告/年度报告/案例总结的自动生成。

| 维度 | 内容 |
|------|------|
| 主模块 | M2 案例与知识沉淀 + M3 传播与运营 |
| 谁来用 | 项目负责人、传播人员、项目督导、其他项目人员 |
| 输入什么 | 项目记录、进展记录、留痕记录、传播推文链接 |
| 输出什么 | 项目进展报告、年度执行报告、案例总结与模式提取、传播匹配度评估 |
| 现有资料 | 50 个项目，存在办公电脑硬盘和企业网盘 |
| 成功标准 | 2 份真实结项报告初稿 2 个工作日内完成；1 位同事愿意继续使用；初稿被资助方认可"结构没问题" |

## 二、📦 课前准备清单

> ⏱ 预计 40 分钟 | 第 1 课前完成

- [ ] **资料盘点**：浏览 50 个项目的文件，按"完整度"分为三档：
  - A 档（项目书+执行+结项三件套齐全）→ 估计有几个？
  - B 档（有项目书和执行记录，缺结项）→ 估计有几个？
  - C 档（只有零散记录）→ 估计有几个？
- [ ] **先拿 A 档开刀**：选 5-8 个 A 档项目，将其资料统一放入 `天津绿邻居_知识库/` 文件夹
- [ ] **准备一份你最近手工写的结项报告**：作为第 1 课的"输出质量参照"
- [ ] **工具确认**：QoderWork 已安装；企业网盘/在线文档可用（知识库云端共享）
- [ ] **想好一个你平时被同事问最多的项目问题**：如"XX 项目的核心产出是什么？"—— 这将成为知识库问答的第一个测试用例

## 三、🎯 最简闭环 MVP

> 第 2 课结束前必须跑通

**MVP 定义**：把 5 个 A 档项目的资料放入文件夹 → QoderWork 建立可搜索索引 → 能通过关键词找到对应项目信息。

**为什么选这个、不先做结项报告**：
- 你的独特需求是"知识库问答"，结项报告是知识库用起来之后水到渠成的产出
- "能搜到"这个体验最有冲击力：把散落在硬盘里的东西变成 5 秒可查的答案
- 先建地基（知识库），再盖房子（报告/案例/评估）

**跑通长这样**：
```
你问："2023 年我们做了哪些社区垃圾分类项目？"
QoderWork 答：
  "从知识库中找到 3 个相关项目：
   1. XX 社区垃圾分类倡导项目（2023.3-2023.12），覆盖 2000 户，产出包括...
   2. ...
   3. ..."
```

## 四、🧩 Skill 模板

### Skill 1：项目知识库搜索（第 1-2 课，核心）

```
【Skill 名称】绿邻居项目知识库
【触发方式】输入 "查 [关键词]" 或 "项目 [项目名称] 详情"
【输入】知识库文件夹（含所有已整理项目资料）
【处理逻辑】
  1. 搜索知识库中匹配关键词的项目
  2. 按相关度排列，返回摘要（项目名/时间/核心产出/状态）
  3. 可追问细节："XX 项目的预算执行情况如何？"
【输出格式】简洁的查询结果（项目列表+一句话摘要）+ 可追问提示
【质检标准】搜索结果覆盖了所有相关项目；摘要准确不误导
```

### Skill 2：结项/年度报告生成（第 3-5 课，基于知识库）

```
【Skill 名称】绿邻居报告生成
【触发方式】输入 "生成结项报告 [项目名]" 或 "生成年度报告 [年份]"
【处理逻辑】搜索知识库中匹配项目/年份的所有资料 → 按报告模板生成
【输出格式】文档初稿
【质检标准】数据与知识库一致；章节结构完整；[待补充] 项不超过 5 处
```

### Skill 3：传播匹配度评估（第 4-5 课，独有需求）

```
【Skill 名称】传播匹配度评估
【触发方式】输入 "评估传播 [活动记录/案例文本]"
【输入】一段活动记录或案例文本
【处理逻辑】
  1. 分析内容特征：情感强度/视觉可呈现性/故事完整性/社会共鸣度
  2. 匹配渠道建议：适合公众号深度/短视频/朋友圈/媒体通稿？
  3. 给出评分（1-5 分）和优化建议
【说明】这是独有需求。建议先在纸上定义评估标准，再让 AI 执行
```

### 复制即用

```
你是一个公益机构项目知识库助手。请按以下规则工作：

1. 当我输入"查 [关键词]"，请搜索知识库文件夹中所有项目资料，
   返回匹配结果，格式如下：
   - 找到 N 个相关项目
   - 每个项目一行：项目名 | 时间 | 核心产出 | 状态
   - 末尾提示：可以继续问"XX 项目的预算执行情况"或"XX 项目覆盖了多少人"

2. 当我输入"生成结项报告 [项目名]"，请：
   - 搜索该项目所有资料
   - 按标准结项报告结构生成初稿（概述/执行/成效/财务/反思）
   - 数据事实原文引用，缺失信息标注 [待补充]

3. 当我输入"生成年度报告 [年份]"，请：
   - 汇总该年所有项目的核心产出和关键数据
   - 不遗漏任一项、不重复统计

4. 搜索优先级的项目名称、年份、核心产出始终放在结果最前面。

现在请先建立搜索能力。输入"查 垃圾分类"测试一下。
```

## 五、🛠️ 分课操作指引

### 第 1 课：建立搜索 → 第一次"搜到"

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：知识库搜索正常 + 3 次测试查询结果 | ✅ 完成标志：能搜到并准确返回项目信息

1. 粘贴"复制即用"提示词 → 告诉 QoderWork 知识库文件夹路径
2. 测试 3 个问题：
   - `查 垃圾分类`（关键词搜索）
   - `项目 XX项目 详情`（指定项目搜索）
   - `2024 年我们做了多少项目？`（统计型查询）
3. 看答案：准确吗？有遗漏吗？有编造的吗？
4. 标注错误 → 修正提示词 → 再测试

### 第 2 课：封装 Skill + 扩展到更多项目

> ⏱ 课上 15 min + 课后 25 min | 📤 提交物：知识库搜索 Skill + B 档项目入库 | ✅ 完成标志：≥20 个项目可被搜索

1. 封装为 Skill，命名为"绿邻居项目百科"
2. 把 B 档项目也加入知识库（分批，每次 5-8 个）
3. 教会 1 位同事使用：输入 `查 [XX关键词]` 即可
4. 收集同事反馈：搜得到吗？答得对吗？还想问什么？

### 第 3 课：结项报告生成

> ⏱ 课上 15 min + 课后 20 min | 📤 提交物：2 份结项报告初稿 | ✅ 完成标志：报告结构被资方认可"没问题"

1. 选 2 个 A 档项目，输入：`生成结项报告 [项目名]`
2. 对比：你手工写的版本和 AI 生成的质量差距？
3. 逐章标注"写得好/需要改/完全不行"
4. 优化提示词 → 封装"结项报告"Skill

### 第 4 课：年度报告 + 传播评估

> ⏱ 课上 15 min + 课后 25 min | 📤 提交物：年度报告初稿 + 传播评估标准表 | ✅ 完成标志：年度报告覆盖了所有已入库项目

1. 输入 `生成年度报告 2025` → 人工校订
2. 定义传播匹配度评估标准（如：情感强度 1-5 / 可视觉化 1-5 / 故事完整性 1-5）
3. 创建"传播匹配度评估"Skill

### 第 5 课：成果预发布 → 知识管家上线

> ⏱ 课上 15 min | 📤 提交物：3 个 Skill 的使用说明 | ✅ 完成标志：至少 1 位同事在日常工作中使用 ≥1 周

1. 打包"绿邻居项目百科"+"报告生成"+"传播评估"三 Skill
2. 给同事群体演示：搜索→查到一个项目→生成它的结项报告→评估传播价值
3. 记录关键数据：搜索平均响应时间 / AI 生成与人工版本的质量对比 / 同事使用频率

## 六、⚠️ 常见坑提醒

| 坑 | 自救方法 |
|----|---------|
| 50 个项目一次全入库后搜索不准确 | 分批入库，每批 5-8 个，入库后立即测试搜索；确保每个项目有统一的关键字段（名称/年份/类型/状态） |
| "传播匹配度评估"太主观，AI 输出不一致 | 先手工定义 3-5 个评估维度的 1-5 分标准，写进提示词，再用已知案例验证评分逻辑 |
| 知识库问答说"找不到"，其实资料在文件夹里 | 检查文件名和内容是否含关键词；要求 QoderWork 先全局扫描生成索引，再执行搜索 |
