# SG-15 · 观鸟综合事务型 · Skill 构建指引（Lite）

> 对应需求代号 T15 | M2 · 知识库问答 + 文书 + 待办清单（最广场景）

---

## 助理画像

| 维度 | 内容 |
|------|------|
| 一句话 | 机构全域资料可检索、可溯源；文书初稿；待办清单表；活动 SOP |
| 主模块 | M2 |
| 输入 | 治理/项目/物种数据/活动/传播/日常事务六类资料 |
| 输出 | 溯源问答、文书、待办表、活动 SOP、协同建议 |
| 成功标准 | 30 份资料入库；15 条待办清单；3 类任务跑通；初稿时长 -80% |

## 最简 MVP（第 2 课结束前）

**30 份核心资料归集** + **1 次检索问答**（问项目/物种/活动任一类，答带来源路径）+ **1 张待办清单表**（15 行固定列）。

## 推荐 Skill

| 顺序 | Skill | 用途 |
|------|-------|------|
| 1 | 自建「待办清单」Skill | 输入散乱事项 → 输出固定列表格 |
| 2 | 知识库检索（第 2 课主能力） | 溯源问答 |
| 3 | [`env-ngo-proposal-draft`](../part-01-skill/skills/env-ngo-proposal-draft/SKILL.md) | 多场景文书 |
| 4 | [`env-ngo-activity-recap`](../part-04-scenario/skills/env-ngo-activity-recap/SKILL.md) | 活动招募/回顾 |

## 待办清单 Skill · 输出契约（复制修改）

```markdown
| 事项 | 来源 | 截止时间 | 负责人代号 | 下一步 |
|------|------|---------|-----------|--------|
```

触发：「整理待办」「本周事项」「仪表盘」→ 本 Skill；「写项目书」→ proposal Skill。

## QoderWork 三步

1. 六类资料分子文件夹，先各放 5 份代表文件。
2. 检索测试问句各 1 条（项目/物种/活动）。
3. 待办 Skill 用秘书提供的 15 条真实事项试跑（脱敏）。

## 分课卡点

| 课次 | 做什么 |
|------|--------|
| 第 2 课 | 30 份入库 + 检索 + 待办表 |
| 第 3 课 | Agent 挂 2 Skill（检索/待办 vs 文书） |
| 第 4 课 | recap 第三 Skill；路由自检必做 |
| 第 5 课 | 三类任务各演示 1 次 |

## 常见坑

- 六类一次全做 → 先 30 份核心 + 3 类任务，再扩展。
- 问答无来源 → 输出必须带文件路径或段落引用。
- 商业活动 Skill 与公益 SOP 混淆 → 触发说明分设。
