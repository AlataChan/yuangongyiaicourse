# SG-10 · 全员双端协同型 · Skill 构建指引（Lite）

## 助理画像

| 维度 | 内容 |
|------|------|
| 一句话 | 全员可用的项目助理：文书 + 宣传 + 答疑 + 案例，先单端跑通再谈双端 |
| 主模块 | M3（兼 M1/M2 能力） |
| 输入 | 工作文本、项目资料、活动素材、协作软件表单导出 |
| 输出 | 标准文书、宣传物料、FAQ、案例、复盘 |
| 成功标准 | L1：元器 Bot 稳定可用，文稿时长 -60%；L2：自有平台+协作软件双端（选修） |

## 最简 MVP（第 3 课结束前）

**元器 Bot + 1 个知识库（≥30 份核心资料）+ 2 个 Skill** 跑通三类任务各 1 次：文书框架、活动文案、资料检索问答。

## 推荐 Skill

| 顺序 | Skill | 用途 |
|------|-------|------|
| 1 | [`env-ngo-proposal-draft`](../part-01-skill/skills/env-ngo-proposal-draft/SKILL.md) 或 closure-report | 文书初稿 |
| 2 | [`env-ngo-community-faq`](../part-04-scenario/skills/env-ngo-community-faq/SKILL.md) | 统一答疑话术 |
| 3 | [`env-ngo-activity-recap`](../part-04-scenario/skills/env-ngo-activity-recap/SKILL.md) | 宣传物料 |
| L2 选修 | 平台 API / 协作软件插件文档 | 双端内嵌，工作坊或 office hour |

## QoderWork 三步

1. 从硬盘+协作网盘**先收 30 份**高频资料到统一文件夹（分批，每批 ≤2GB）。
2. 第 2 课建知识库索引；定义 10 条标准测试问句。
3. 第 3 课挂载到元器；**L1 验收到此为止**。

## 分课卡点

| 课次 | 做什么 |
|------|--------|
| 第 2 课 | 知识库分批入库 + 检索测试 |
| 第 3 课 | Agent 编排 + 三类任务冒烟测试 |
| 第 4 课 | 第二 Skill + 路由边界（文书 vs FAQ vs 推文） |
| 第 5 课 | 内测：≥3 位同事各完成 1 次真实任务 |
| 工作坊（L2） | 双端集成方案评审，不纳入 5 课必修 |

## 常见坑

- 一上来做 API 集成 → 先 L1 元器跑通再扩展。
- 1000 条资料一次入库 → 先 30 份核心，验证再扩。
- Skill 过多路由冲突 → 第 4 课必做路由自检表。
