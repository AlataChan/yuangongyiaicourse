# SG-14 · 湿地项目管理型 · Skill 构建指引（Lite）

> 对应需求代号 T14 | M2 · OKR 清单 + 案例 + 社群答疑

---

## 助理画像

| 维度 | 内容 |
|------|------|
| 一句话 | 项目书 → OKR/里程碑表；案例记录 → 经验报告；社群湿地知识答疑 |
| 主模块 | M2 |
| 输入 | 项目书、目标/节点、财务流水摘要、活动记录 |
| 输出 | OKR 表、进度提醒清单、案例集、社群 FAQ |
| 成功标准 | 2 个项目 OKR 表 + 每周案例报告 + 社群答疑可用 |

## 最简 MVP（第 2 课结束前）

**1 份项目书** → Skill 输出 OKR/里程碑 Markdown 表（目标/关键结果/负责人/截止）→ 人工确认。

## 推荐 Skill

| 顺序 | Skill | 用途 |
|------|-------|------|
| 1 | 自建「项目书→OKR 表」Skill（Fork proposal-draft 改输出契约） | 结构化清单，非 BI 系统 |
| 2 | [`env-ngo-case-extract`](../part-04-scenario/skills/env-ngo-case-extract/SKILL.md) | 活动记录 → 案例卡 |
| 3 | [`env-ngo-community-faq`](../part-04-scenario/skills/env-ngo-community-faq/SKILL.md) | 湿地保护社群答疑 |
| 4（可选） | closure-report 变体 | 财务/进度段落模板 |

## QoderWork 三步

1. `项目管理/` 放 2 个项目书 + 飞书/云盘导出摘要。
2. OKR Skill 输出存 `进度清单/`；案例输出存 `案例/`。
3. 元器定时提醒：用 Bot 提醒功能 + 清单 Markdown，不做独立仪表盘产品。

## 分课卡点

| 课次 | 做什么 |
|------|--------|
| 第 1 课 | OKR 表 Skill（固定列） |
| 第 2 课 | 项目资料入库 |
| 第 4 课 | case-extract + FAQ 第二、三 Skill |
| 第 5 课 | 演示：OKR 表 + 1 份案例 + 3 条 FAQ |

## 常见坑

- 「仪表盘」误解为软件 → 交付物是**固定格式 Markdown 表** + 提醒。
- 财务数字 AI 核算 → 只生成段落框架，数字人工核对。
