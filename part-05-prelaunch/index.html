成果盘点
让同事真正用起来

前四课你已经有了技能、知识库和 AI智能体。但「我能用」和「同事真在用」之间，还差最后一公里。 这节课带你盘点产出、完成预发布演示、落实上线方案——线上课的收官之课。

时长 90 分钟 产出 预发布演示 + 发布计划 从「我能用」到「同事真在用」
打开学员手册 查看三层盘点

课后自测：课后测验（10 题）→ 覆盖本课 3 项教学目标（三层盘点 / 预发布与发布方案 / 优化节奏），每题标注对应《学员手册》章节。

5.1 成果盘点 · 25 分钟

三层产出盘点

拿出课前作业填写的立项书，逐层核对——技能、知识库、智能体，每一层都有明确的达标线。

🛠

技能层

达标线：≥2 个技能跑通

触发说明清晰
输出契约完整
Gotchas ≥6 条
📚

知识库层

达标线：有量有质

文档数量达标
切片质量抽检通过
检索准确率可接受
🤖

智能体层

达标线：主流程可跑通

正确路由到技能与知识库
不编造、不越界
路由测试 ≥5 条用例
L1/L2 分层自评：L1（用起来）= ≥1 技能 + 知识库可检索 + Agent 能回答基本问题；L2（用得顺）= ≥2 技能 + 覆盖主场景 + 同事能独立使用。未达标不等于失败——写下差距，带到工作坊补齐。
5.2 预发布演示 · 35 分钟

每机构 5 分钟，跑一遍真实场景

展示你的主工作流，回答一个现场真实问题——不是排练，是实战。

演示格式

  • 展示主工作流：从输入到最终输出，用真实材料跑一遍完整流程
  • 回答 1 个真实问题：由现场同学即兴提问，观察 Agent 的应答质量
✅

可用性

同事拿到能直接用吗？操作步骤是否清晰？

🎯

准确性

回答是否基于知识库？有没有编造内容？

📢

推广意愿

你愿意推荐给自己的同事用吗？

5.3 发布与日常化预案 · 20 分钟

发布渠道与日常化

确定你的 Agent 从哪个渠道触达同事和用户——内部先行，对外按需。

💬

飞书 / 企微机器人

内部日常问答、文书辅助，同事在工作群里直接 @机器人 即可使用。

内部渠道
📱

公众号自动回复

对外常见问题自动应答，适合 M3 阶段面向公众与志愿者的场景。

M3 对外
🌐

Web 嵌入（官网）

嵌入机构官网，为外部访客提供项目咨询与信息查询服务。

官网嵌入

上线检查清单（预览）

技能触发说明已更新到最终版
知识库文档已完成脱敏审核
Agent 路由测试通过（≥5 条用例）
敏感信息兜底规则已设置
指定 1 位「日常维护人」
确定首批试用同事（≥2 人）
准备好 2 周效果评估卡
90 天行动计划

30 / 60 / 90 天，用数据证明价值

不只是上线，更要跑起来——每个阶段都有明确的目标和数据回收点。

第 30 天
试运行：小范围跑通、收集真实反馈
  • 完成首批 2-3 位同事试用
  • 填写 2 周效果评估卡（使用次数、准确率、满意度）
  • 根据反馈修正 ≥1 个技能的 Gotchas
  • 知识库补充 ≥5 份新文档
第 60 天
扩大使用：从「试用」变成「日常工具」
  • 试用范围扩大到全机构
  • 累计替代手工工作 ≥10 次
  • 新增 ≥1 个技能（覆盖第二场景）
  • 建立知识库更新频率（如每月 1 次）
第 90 天
数据回收 + 复盘：用数据决定下一步
  • 填写 90 天效果总表
  • 计算累计节省工时
  • 同事满意度调查
  • 决定是否扩展到对外场景（公众号 / 官网）
  • 输出 1 份「AI 助理运行报告」给管理层
