## 一、这节课要解决什么

前四课你已经有了技能、知识库、AI智能体，也按主场景做了深化。但「我能用」和「同事真在用」之间，还差最后一公里。

第 5 课带你完成三件事：盘点产出是否达标、在同伴面前跑一遍真实演示、把发布渠道与日常化预案落到纸上。下课时，你手上应该有一份可以直接执行的上线方案。

这是「技能 → 知识库 → AI智能体」主线的收官环节，线下工作坊将在此基础上做跨机构互换与正式部署。

## 三、5.1 成果盘点（25 分钟）

### 三层产出对照表

拿出课前作业填写的**立项书**，逐层核对：

| 层级 | 达标线 | 检查项 |
|------|--------|--------|
| 技能层 | ≥2 个技能跑通 | 触发说明清晰、输出契约完整、Gotchas ≥6 条 |
| 知识库层 | 知识库有量有质 | 文档数量达标、切片质量抽检通过、检索准确率可接受 |
| 智能体层 | 主流程可跑通 | Agent 能正确路由到技能与知识库、不编造、不越界 |

### L1 / L2 分层自评

| 层级 | 标准 | 你的状态 |
|------|------|---------|
| L1 入门（用起来） | ≥1 技能跑通 + 知识库可检索 + Agent 能回答基本问题 | ☐ 达标 / ☐ 未达标 |
| L2 进阶（用得顺） | ≥2 技能 + 知识库覆盖主场景 + Agent 路由准确 + 同事能独立使用 | ☐ 达标 / ☐ 未达标 |

> 未达标不等于失败——把差距写下来，带到线下工作坊补齐。

## 四、5.2 预发布演示（35 分钟）

### 演示格式

每个机构 **5 分钟**，包含：

```text
演示内容：
1. 展示主工作流（从输入到最终输出，跑一遍真实任务）
2. 回答 1 个真实问题（现场提问，不是预设的）
```

### 同伴互评维度

观众从三个维度打分并给出一句话建议：

| 维度 | 关注点 | 评价 |
|------|--------|------|
| 可用性 | 同事拿到能直接用吗？操作步骤是否清晰？ | ★☆☆ ~ ★★★ |
| 准确性 | 回答是否基于知识库？有没有编造？ | ★☆☆ ~ ★★★ |
| 推广意愿 | 你愿意推荐给自己的同事用吗？ | ★☆☆ ~ ★★★ |

## 五、5.3 发布与日常化预案（20 分钟）

### 三条发布渠道

| 渠道 | 适用场景 | 触达对象 | 上线难度 |
|------|---------|---------|---------|
| 飞书 / 企微机器人 | 内部日常问答、文书辅助 | 机构同事 | 低 |
| 公众号自动回复 | 对外常见问题自动应答（M3 阶段） | 公众、志愿者 | 中 |
| Web 嵌入（官网） | 嵌入机构官网，提供项目咨询 | 外部访客 | 中高 |

### 上线检查清单

```text
上线前逐项确认：
- [ ] 技能触发说明已更新到最终版
- [ ] 知识库文档已完成脱敏审核
- [ ] Agent 路由测试通过（≥5 条用例）
- [ ] 敏感信息兜底规则已设置（拒答 / 脱敏）
- [ ] 指定 1 位「日常维护人」
- [ ] 确定首批试用同事（≥2 人）
- [ ] 准备好 2 周效果评估卡
```

### 2 周效果评估卡

试用 2 周后填写：

| 指标 | 数据 |
|------|------|
| 实际使用次数 | ___ 次 |
| 替代手工工作时长 | 约 ___ 小时 |
| 回答准确率（抽检 5 条） | ___% |
| 同事满意度（1–5） | ___ 分 |
| 需要人工修正的比例 | ___% |
| 最常见的失败场景 | ____________ |

## 七、L1 / L2 分层自评表

### L1 入门（用起来）

```text
自评项目：
- [ ] 有 ≥1 个可跑通的技能
- [ ] 知识库已上传核心文档（≥10 份）
- [ ] Agent 能回答所选场景的基本问题
- [ ] 跑通过 ≥1 次真实任务替代

```

### L2 进阶（用得顺）

```text
自评项目：
- [ ] 有 ≥2 个技能（含 1 个文书链）
- [ ] 知识库覆盖主场景，检索准确率可接受
- [ ] Agent 路由准确，不误触发
- [ ] ≥3 次真实任务替代，有时间对比记录
- [ ] 同事能独立使用，无需你在旁边盯着
- [ ] 上线检查清单已完成
```

## 八、90 天行动计划

### 30 天（试运行）

```text
目标：小范围跑通、收集真实反馈
- [ ] 完成首批 2–3 位同事试用
- [ ] 填写 2 周效果评估卡
- [ ] 根据反馈修正 ≥1 个技能的 Gotchas
- [ ] 知识库补充 ≥5 份新文档
```

### 60 天（扩大使用）

```text
目标：从「试用」变成「日常工具」
- [ ] 试用范围扩大到全机构
- [ ] 累计替代手工工作 ≥10 次
- [ ] 新增 ≥1 个技能（覆盖第二场景）
- [ ] 知识库更新频率建立（如每月 1 次）
```

### 90 天（数据回收 + 复盘）

```text
目标：用数据证明价值、决定下一步
- [ ] 填写 90 天效果总表
- [ ] 计算累计节省工时
- [ ] 同事满意度调查
- [ ] 决定是否扩展到对外场景（公众号 / 官网）
- [ ] 输出 1 份「AI 助理运行报告」给管理层
```
